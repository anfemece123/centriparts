# PROJECT_CONTEXT.md

## Product summary
This project is a complete ecommerce platform.

## Initial scope
The first version should include:
- authentication
- admin panel
- categories
- products
- product images
- product variants
- inventory
- orders
- customers
- storefront
- cart
- checkout

## Technical direction
The project uses Supabase as the backend platform:
- PostgreSQL database
- authentication
- storage
- realtime if needed
- edge functions later if needed

## Frontend direction
The frontend must use:
- React
- TypeScript
- Redux Toolkit
- scalable modular architecture

## Important priorities
- strong architecture from the beginning
- maintainable codebase
- clear folder structure
- domain-driven thinking
- easy future expansion

## Language rules

The codebase must be written in English.

However, the user interface of the application must be entirely in Spanish.

All visible text for users should be Spanish.

Use a translation-based structure to keep UI text organized.

## UI language
All user-facing interface text must be in Spanish.

## Visual direction
The application must use:
- white as the main background color
- yellow as the primary brand color
- black for contrast and strong accents

The visual style must feel:
- professional
- organized
- modern
- trustworthy
- clean
- ecommerce-focused

## UI implementation
Use Tailwind CSS for styling.

The interface should avoid visual clutter and should prioritize:
- clear navigation
- strong visual hierarchy
- clean spacing
- reusable components
- polished forms
- professional tables
- clear cards and content sections

## Visual inspiration
The visual inspiration is a professional ecommerce style similar to established commercial catalog/store websites.

Important inspiration points:
- visible search
- clear access to account and cart
- organized category navigation
- strong commercial feel
- clean and trustworthy layout

Do not copy any existing site literally.
Use the inspiration only for layout quality, organization, and professionalism.
